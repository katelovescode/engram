TV review redesign — direction mockups

Four static directions for the disc-review screen, all built on the same data and the same hard case: making it easy to manually correct a TV disc when the matches are interdependent. Pick one (or a hybrid) and we'll build it. Nothing here is wired — they're visual references.

The seed scenario (identical in all four)

Disc
For All Mankind · Season 3 · 6 ripped titles
Confident
Titles 1,2,5,3 → E01, E02, E03, E06 (90–96%)
Conflict
Title 6 and Title 4 both land on E05 — Title 6 is stronger (88% vs 54%)
Unmatched
Title 4 is low-confidence; its top guess (E05) is taken
The gap
E04 — Happy Valley is the only empty slot, and it fits Title 4's runtime
The fix
Move Title 4 off E05 and into E04 — a whole-disc reconciliation, not a single edit

Shared data & feature layer

  • Ranked candidates with evidence — each shows episode code + name, score %, votes (x/10), coverage %, and the top match's score gap ("decisive" vs "uncertain"). Already computed by the matcher.
  • Episode names — pulled from a new season-roster endpoint (codes → titles), so you never decide on a bare "S03E05".
  • Season roster + conflict/coverage — the season's episodes marked assigned / duplicated / missing across the disc, with the obvious gap suggested for an unmatched title.
  • Provenance — engram / discdb / manual source and any DiscDB review flag.
